## What was fixed/hardened along the way

- **arXiv fetcher**: `export.arxiv.org` now 301-redirects to HTTPS; the fetcher
  used HTTP and followed nothing, returning 0 papers. Switched to HTTPS +
  `follow_redirects` ([growthos/sources/arxiv.py](growth-os/growthos/sources/arxiv.py)).
- **Signals scoring**: HN/blog headlines are too short to match the multiword
  interest profile, so every signal scored 0 and was dropped. Signals now keep
  zero-scored items (the feed list itself is hand-picked = already relevant);
  only penalized items are dropped. Papers/repos keep the positive-score gate
  ([growthos/score.py](growth-os/growthos/score.py)).
- **Live write path verified** (was the flagged unknown): rows upsert via
  `PUT /api/workspace/{ws}/database/{db}/row` with `pre_hash=external_id` —
  the server hashes it into the row id, so re-sends update instead of
  duplicate. Cells go over the wire keyed by **field id**; the name→id map
  lives in [growth-os/config/databases.json](growth-os/config/databases.json)
  (generated by [scripts/setup_workspace.py](growth-os/scripts/setup_workspace.py)).
- **Selects**: option values only persist if the option exists on the field, so
  the setup script pre-creates all schema options, and free-form tag columns
  (Topics/Tags/Habits) became plain text in
  [config/schema.yaml](growth-os/config/schema.yaml).
- **User model**: AppFlowy's free self-host tier allows 1 user; the GoTrue
  admin is a *system* account that cannot own a workspace. So
  `admin@example.com` is the admin-console account and your gmail is the real
  AppFlowy user (same password, see CREDENTIALS.md).
- **Brief engine**: added `python -m growthos.brief` — reads the last day of
  papers/repos/signals + due lessons + next book from AppFlowy, renders
  `_export/brief_YYYY-MM-DD.md`, and upserts one row per day into `review`.

## What still needs you (can't be automated from here)

1. **Tailscale (phone access)** — not installed on this machine. Install on
   the PC (`winget install tailscale.tailscale`), sign in, then install the
   Tailscale + AppFlowy apps on your phone and point AppFlowy at
   `http://<this-pc-tailnet-name>` . Then update `FQDN` in
   `AppFlowy-Cloud\.env` to that name and `docker compose -p appflowy up -d`
   to re-issue the right URLs.
2. **Desktop/mobile app sign-in** — in the AppFlowy app: Settings → Server →
   AppFlowy Cloud Self-hosted → `http://localhost` (or the tailnet URL), then
   log in with your gmail + password.
3. **GitHub PAT** (optional, lifts the rate limit that 403'd one query):
   put it in `growth-os\.env` as `GITHUB_TOKEN=` and
   `docker restart growthos-curator`.
4. **Seed `library`** — import your `curriculum-appflowy.csv` in the UI, or
   ask Claude to do it through the row API.
5. **MCP for Claude** — `growth-os/agent/mcp.config.json` is pre-filled with
   your real values; merge it into your Claude config when you pick an
   AppFlowy MCP server (community ones exist; read their source first). For
   Claude Code in this repo, the venv + `growthos.appflowy.AppFlowyClient`
   already gives full read/write access without any MCP.

## Useful commands

```powershell
# tail the curator loop
docker logs growthos-curator --tail 20
# run the curator / brief by hand
cd growth-os; .venv\Scripts\python.exe -m growthos.curate
cd growth-os; .venv\Scripts\python.exe -m growthos.brief
# stop / start everything
docker compose -p appflowy stop;  docker compose -p appflowy start
docker stop growthos-curator;     docker start growthos-curator
# backups (run on a schedule you like)
docker exec appflowy-postgres-1 pg_dump -U postgres postgres > appflowy_backup.sql
```

Tuning lives in one file: `growth-os\config\sources.yaml` (interest weights,
penalties, arXiv categories, GitHub queries, RSS feeds, top-N per source).
The curator picks changes up on its next hourly run (config is volume-mounted).
